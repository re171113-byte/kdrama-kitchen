## 1. Pinterest 알고리즘 핵심

**Pinterest는 소셜 미디어가 아니라 비주얼 검색 엔진이다.**

Instagram/X와 달리 팔로워 수보다 **검색 적합성**이 중요하고, 콘텐츠 수명이 24~48시간이 아니라 **몇 주~몇 달**이다.

### 핵심 랭킹 요소

| 요소 | 설명 |
|------|------|
| **신선도** | 새 이미지 + 새 URL이 가장 높은 점수. 같은 URL이라도 새 이미지면 "신선한 핀" |
| **키워드 적합성** | 핀 제목, 설명, 보드 제목, alt text의 키워드를 분석 |
| **저장 수 (Save)** | 가장 가치 있는 인게이지먼트 신호. 저장 비율 높으면 배포 확대 |
| **클릭률** | 핀 → 사이트 클릭이 높으면 알고리즘이 더 넓게 배포 |
| **이미지 품질** | 고해상도, 세로형, 텍스트 오버레이 등 시각적 품질 분석 |
| **도메인 신뢰도** | 링크된 사이트의 품질 점수. 다른 사용자가 해당 사이트를 핀하면 점수 상승 |
| **보드 관련성** | 핀이 저장된 보드의 주제와 핀 내용이 일치해야 함 |

### 노출 과정

```
1. 새 핀 등록 → Pinterest가 이미지/텍스트/키워드 분석
2. 관련 검색 결과 + 홈 피드에 소규모 노출
3. 저장/클릭 반응 좋으면 → 더 넓은 배포
4. 시간이 지나도 검색으로 계속 노출 (장기 트래픽)
```

## 2. 핀 이미지 규격

### 필수 사양

| 항목 | 규격 |
|------|------|
| **비율** | 2:3 (세로형) |
| **해상도** | 1000 × 1500 px (최소) |
| **파일 형식** | JPG 또는 PNG |
| **파일 크기** | 5MB 이하 권장 (최대 20MB) |

### 텍스트 오버레이 규칙

```
- 짧고 키워드가 포함된 텍스트 (예: "Korean Mandu Recipe")
- 굵고 읽기 쉬운 폰트
- 한 가지 메시지만 (복잡하면 가독성 하락)
- 핀의 가치 제안을 명확히 (왜 저장해야 하는지)
- 모바일에서 잘 보이는 크기
```

### 비주얼 디자인 원칙

```
- 밝고 대비 높은 색상
- 음식 사진: 선명하고 식욕을 자극하는 앵글
- 브랜드 일관성: 같은 폰트, 색상, 로고 위치
- CTA 포함 (예: "Save for later", "Full recipe →")
```

## 3. 핀 SEO (키워드 최적화)

### 키워드를 넣어야 하는 곳

| 위치 | 설명 | 글자 제한 |
|------|------|----------|
| **핀 제목** | 가장 중요한 랭킹 요소. 주요 키워드를 앞에 배치 | 100자 |
| **핀 설명** | 자연스러운 키워드 사용. 220~232자가 최적 | 500자 |
| **보드 제목** | 니치 구체적 (예: "K-Drama Korean Recipes") | - |
| **보드 설명** | 관련 키워드 포함 | - |
| **Alt Text** | 이미지 설명 + 키워드 | - |

### 제목 작성법

```
좋은 예: "Doctor Slump Mandu — Classic Korean Dumplings Recipe"
나쁜 예: "Mandu Recipe"

구조: [드라마명] [음식명] — [구체적 설명]
```

### 설명 작성법

```
구조:
  1줄: 드라마 맥락 + 이모지 (hook)
  2~3줄: 음식 설명 + 핵심 포인트
  4줄: 레시피 하이라이트 (리스트)
  5줄: CTA ("Full recipe + video tutorial")
  마지막: 해시태그 8~10개

글자수: 220~232자 최적 (키워드 스터핑 금지)
```

### 해시태그 전략

```
필수 포함:
  #KoreanFood #KDrama #KoreanRecipe #[드라마명] #[음식명]

선택 추가:
  #ComfortFood #KoreanCooking #[음식영문명] #[관련카테고리]

총 8~10개, 관련성 높은 것만
```

## 4. 포스팅 전략

### 핀 구조 (레시피 포스트당)

```
제목: "[Drama Name] [Food Name] — [Subtitle]"
설명: 드라마 맥락 + 레시피 하이라이트 + 해시태그 (220~232자)
이미지: 2:3 세로형, 텍스트 오버레이 포함
링크: 레시피 페이지 URL
보드: K-Drama Recipes
태그: 5개
```

### 배포 규칙

```
- 같은 날 2개 동시 게시 가능 (Pinterest는 스팸 감지 덜 엄격)
- K-Drama Recipes 보드에 먼저 저장 (첫 보드가 주제 신호)
- 일관성 유지: 매주 3~5개 품질 핀 (대량 핀 X)
- 같은 URL이라도 새 이미지면 새 핀으로 다시 게시 가능
```

### 이미지 제작 흐름

```
1. 원본 음식 사진 준비 (고해상도)
2. 2:3 비율 (1000×1500px) 크롭
3. 텍스트 오버레이 추가 (드라마명 + 음식명)
4. 브랜드 일관성 확인 (폰트, 색상)
5. 파일 크기 5MB 이하 확인
6. temp-images/ 에 저장
```

## 5. 자동화 스크립트 구조

### 접속 방식

```javascript
const { chromium } = require('playwright');
const PROFILE_DIR = 'C:/Users/noble/ChromeX';

const context = await chromium.launchPersistentContext(PROFILE_DIR, {
  channel: 'chrome',
  headless: false,
  args: ['--disable-blink-features=AutomationControlled'],
  viewport: { width: 1280, height: 800 },
});
const page = await context.newPage();
```

- Playwright `launchPersistentContext` + ChromeX 프로필
- 프로필에 Pinterest 로그인 상태 유지
- `--disable-blink-features=AutomationControlled`로 봇 감지 우회

### 스크립트 실행 흐름

```
1. ChromeX 프로필로 브라우저 실행
2. Pinterest 핀 생성 페이지 이동
3. 이미지 업로드 (input[type="file"])
4. 제목 입력 (#storyboard-selector-title)
5. 설명 입력 (Tab → type)
6. 링크 입력 (#WebsiteField)
7. 보드 선택 (K-Drama Recipes)
8. 게시 버튼 클릭
9. 게시 확인
```

## 6. 향후 업그레이드 가능 항목

- **Idea Pins (멀티 페이지)**: 일반 핀 대비 4x 인게이지먼트
- **비디오 핀**: 15~60초, 9:16 비율 (1080×1920px)
- **Rich Pins**: 레시피 메타데이터 자동 표시 (사이트에 Schema 추가 필요)
- **도메인 인증**: Pinterest에서 사이트 도메인 인증 → 신뢰도 상승
